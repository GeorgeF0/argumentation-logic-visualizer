Argumentation Logic Visualizer Help Page

Welcome to the Argumentation Logic Visualizer tool! Below you can find help on how to use this program and its features

Clipboard

The Clipboard holds proofs and other items that need to be retained. It acts as storage but also acts as a pool of available items to use on the Workbench.

Thumbnails

Each item is rendered in its own thumbnail, and can be dragged to its appropriate placeholder on the Workbench or on the Clear Clipboard button described below. A thumbnail looks like this:

This is a thumbnail!

Drag/Drop Functionality

As mentioned before, thumbnails can be dragged around and dropped on particular placeholders on the Workbench, or on the Clear Clipboard button. This button looks like this:

Dropping an item from the Clipboard onto this button will delete the item from the Clipboard. Clicking this button will delete all the contents of the Clipboard, but after a warning pop-up appears. There is no warning pop-up for dragging a single item onto this button and that item is deleted immediately.

A thumbnail can also be dragged and dropped onto the Export Clipboard button. This will open a new window with the exported proof inside. For more information on this refer to the last section of this help page. Clicking on the Export Clipboard button will export the entire Clipboard. The Export Clipboard button looks like this:

Storage

This application uses the HTML 5 web storage feature in order to save the contents of the Clipboard in case the web page is refreshed or closed. However, this storage may be deleted if the browser clears its data, so it should not be heavily relied upon. There is an export clipboard feature that can export the items stored in the Clipboard.

The web storage keeps the Clipboard in sync with itself. Whenever an item is added or deleted, or when the Clipboard is cleared using the Clear Clipboard button, the web storage is updated accordingly. The data is automatically loaded back into the clipboard when the page is refreshed.

Workbench

The Workbench is where the magic happens. The different tasks that can be performed have been split into tabs that all fall under the Workbench. Some of these tasks require the user to input pre-made proofs or arguments from the Clipboard, or to input logic sentences or natural deduction steps. The syntax used for logic sentences and natural deduction is explained in the next section

If such a specific proof is required, it can be typed in directly instead of generating it and finding it in the list of all generated proofs. The theory will then appear as the first steps of the proof and the command input field is enabled and ready to take commands. The commands will be explained shortly. Commands

In order to enter a command, simply type in the command in the command input field and press the Enter key on the keyboard.

The following commands are supported:

  • Step
  • Delete last step
  • Delete last n steps

A step command is of the form "[formula]; [reason]". These contructs are described in the penultimate section. This command is used to add another step to the proof

The delete last step command is simply "--". This will delete the last step of the proof. If the last step of the proof was an application of the not introduction rule, then the context opens the box that the not introduction application had previously closed.

The delete last n steps command works much like the previous command, but the format is "-[number]".

The proof builder uses a parser and a rule verification system in order to check that the proof is according to the rules defined in natural deduction. If an incorrect command is entered, then the command input field will turn red as shown below:

Simply amend the command and try again.

Logic Syntax

This section introduces the syntax used for propositional logic, and how a natural deduction proof is represented.

Symbols for Propositional Logic

The symbols for propositional logic used by this application are:

Symbol Meaning
! Negation
& Conjunction
| Disjunction
-> Implication
_ Falsity (Contradiction)
() Precedence (Grouping)

The same symbols are used by the parser as well.

Entering Propositional Formulas

Single propositional formulas can be entered by typing in the formula using the symbols shown above. This is usually required when entering a goal to be proven. A list of propositional formulas is a comma-separated list of formulas. Space characters are ignored. Lists of formulas are usually required when specifying the theory (which might consist of more than just one formula).

Natural Deduction Format

Each line (step) in a natural deduction proof is made of the line number of that particular step, the deduction made during that step, and lastly, the reason for that conclusion. The reason may be a hypothesis (which makes subsequent steps enter a new context until a matching not introduction rule closes the box), a given (used to indicate the theory) or an application of a rule. A rule is made of a propositional logic symbol (for example "&"), the letter "E" or "I" (meaning "Elimination" or "Introduction") and a pair of parentheses in which line numbers referencing previous steps are given to justify the rule (for example "(1,4)" - read "from lines 1 and 4"). Sub-contexts (boxes) are indicated by indentation. A hypothesis always opens a box (starts a new context) and an application of the not introduction rule or the arrow (implication) introduction rule close the box.

A short summary of all justifications (reasons) found in a natural deduction proof is given below. The symbol "#" just indicates any number referencing a previous step in the proof.

Justification Meaning
!I(#,#) Not Introduction
!E(#) Not Elimination
&I(#,#) And Introduction
&E(#) And Elimination
|I(#) Or Introduction
|E(#,#,#,#) Or Elimination
->I(#,#) Arrow Introduction
->E(#,#) Arrow Elimination
_I(#,#) Falsity Introduction
_E(#) Falsity Elimination
given Indicates theory
hypothesis Hypothesis
check(#) Reiteration

The importing-exporting facility is described below:

Exporting Clipboard

The data stored in the Clipboard can be exported as strings of JSON text and can be saved/distributed/etc. Clicking on the Export Clipboard drop-down menu (or button on the Clipboard) will export the entire contents of the Clipboard. The data will be opened in a new window. If only one of the items on the Clipboard needs to be exported, simply drag that item onto the Clipboard's Export Clipboard button. Note that exporting items on the Clipboard does not delete them from it.

Importing to Clipboard

The text exported by the Export Clipboard drop-down menu or button can be imported back into the Clipboard by clicking on the Import to Clipboard drop-down menu. A pop-up appears with an input field that allows you to paste the JSON text in. Clicking on the Import to Clipboard button will validate the pasted text and if the validation succeeds, the parsed data will be added back to the Clipboard. If the validation fails however, the input field will turn red. Fix the errors and try again.
